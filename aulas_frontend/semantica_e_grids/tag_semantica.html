header

header define o cabeçalho para a página ou outros containers (pode ser usado várias vezes).

main

main ficará todo o conteúdo principal da página, sendo não recomendado o uso de mais de um elemento main.

section

section define uma seção de conteúdo, que pode conter um título e outros elementos. Não é recomendado usar section para conteúdo que não seja uma seção lógica (para conteúdo mais genérico é recomendado usar a tag div).

article

article define um conteúdo independente que pode ser distribuído ou reutilizado de forma independente, como um post de blog, uma notícia ou um comentário.

O elemento article é ideal para conteúdos que podem ser lidos e compreendidos de forma isolada, sem depender do contexto da página.

aside

aside define um conteúdo relacionado, mas não essencial ao conteúdo principal da página. É frequentemente usado para barras laterais, links relacionados ou informações adicionais.

O elemento aside pode conter informações que complementam o conteúdo principal, mas que podem ser lidas de forma independente.

nav

Define um container que deve agrupar links de navegação (não é obrigatório que seja para todas as tag a).

Normalmente é usado para menus de navegação, links em rodapé e entre outros.

footer

footer define o rodapé da página ou de outros containers (pode ser usado várias vezes).

address

usado para definir informações de contato (endereço, email, telefone, url, etc).
